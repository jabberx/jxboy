Description of the machine

1024 bytes RAM (first 45 bytes are syste: first byte is 8-key keyboard bitmask, next 32 bytes is 16x16 display bitmask, see system bytes description below)

1024 bytes stack

2048 bytes call stack (1024 return entries)

ROM size: up to 65536 bytes

No registers but RAM cell shortcut system is present. One RAM cell pointer.

Around 32KHz 8bit CPU

Machine codes

82 possible codes, each code is 1 byte and is printable character.

w.^y"',_x$<>[]z=+-*/|&~%?!:#`@;{}()0123456789abcdefghijklmnopqrstuv\IDLRABCJKFMNOG

This includes operations with stack (^y"',_x), simple 8-bit arithmetic (+-/*),

logical operators (&|~%) (% is xor here), copy/swap commands ({}()), far/short calls (@`),

far/short jumps (#:), return using call stack (;), conditional z/nz jumps (?!),

ram pointer movement (<>[]=) and setting values of ram cells ($\0123456789abcdefghijklmnopqrstuv).

ram pointer shortcut system (ABCJK NOG F NOG M)

Detailed description:

w nop

. halt

^ push

y pop

" dup (stack)

' drop (stack)

, swap (stack)

_ clear stack

x reverse stack

< move data pointer left

> move data pointer right

[ decrease data pointer step /=2

] increase data pointer step *=2

z reset data pointer step to 1

= data ptr = stack.peek(0):stack1.peek(1)

+ add right, wrap, set O to 1 if overflow

- sub right, wrap and set N to 1 if negative

* mul with right, high byte will go to O

/ div to right, remainder to G

| or to right

& and to right

~ not

% xor to right

? short jump if zero to 1sb relative offset in a stack

! short jump if not zero to 1sb relative offset in a stack

: unconditional short jump to 1sb relative offset in a stack

for example g0 on some cell will make it 0b00000000,

i on same cell after that will turn it into 0b00100000

\ push program pointer address to a stack (low then high)

$ push data pointer address to a stack

I increase by 1

D decrease by 1

L left shift (1), O will be set to 0 or 1 (if overflow)

R right shift (1)

Memory pointer shortcuts:

N use negative flag byte intead of RAM pointer

O use overflow flag byte instead of RAM pointer

G use division additional result byte instead of RAM pointer

A use addr:1+32+5 instead of RAM pointer

B use addr:1+32+7 instead of RAM pointer

C use addr:1+32+9 instead of RAM pointer

J use addr:1+32+11 instead of RAM pointer

K use addr:1+32+13 instead of RAM pointer

M restore using RAM pointer

F set RAM pointer to addr:1+32+17 (first unused byte)

RAM layout:
input byte (1) - video bytes (32) - neg byte (1)
- overflow byte (1) - quotient after dot byte (1)
- A shortcut (1) - empty (1) - B shortcut (1) - empty (1)
- C shortcut (1) - empty (1) - J shortcut (1) - empty (1)
- F shortcut (1) - empty(979)
